# 知识秘书 Agent — 日常接入指南

> 目标：像 Skill 一样，开箱即用，不需要主动切换工具。

---

## 核心思路：三种接入姿势

```
浅度接入（推荐入门）
  └─ Box/WorkBuddy：每次对话自动激活，什么都不用改
  
中度接入（开发场景）
  └─ Claude Code CLI：在项目目录放 CLAUDE.md，进目录自动激活
  
深度接入（完全融合）
  └─ 定时任务 + 企微推送：不用打开任何工具，知识自动找到你
```

---

## 方式一：Box 对话 — 最简，一次配置永久生效

### 一次性安装
```
use_skill("https://github.com/Yanyeoo/inspiration-workflow-skill")
```

### 之后的日常——就是正常说话

你不需要说任何特殊命令。Skill 在后台监听，识别到有价值的内容自动处理：

```
你：刚看了一篇关于 GraphRAG 的论文，感觉可以用在知识库项目里

知识秘书：✅ 已记录到 [知识库项目]：GraphRAG 论文（paper）
          标签：#GraphRAG #知识库

你：（继续正常工作，什么都没有被打断）
```

**关键**：你不需要切换模式，不需要说"记一下"，Skill 就是你的对话环境本身。

---

## 方式二：Claude Code CLI — 进项目目录自动激活

### 配置（一次性）
```bash
# 安装后，在你的主要项目目录放一个软链
cd ~/your-main-project
ln -sf ~/.claude/commands/inspiration-workflow.md CLAUDE.md
```

### 日常使用
```bash
cd ~/your-main-project
claude   # ← 进入目录，CLAUDE.md 自动被读取，知识秘书就在那里

# 正常写代码，遇到想法直接说：
> 发现这里用责任链模式比 if-else 更好，记一下

# 遇到 bug：
> 卡住了，Redis 连接池在高并发下有死锁
→ 知识秘书记录 blocker + 搜索知识库 + 推荐解法
```

**效果**：每次进入项目目录启动 claude，知识秘书自动在场，不需要任何额外操作。

## 方式三：定时任务 — 知识主动找到你（最"自动"的姿势）

这是真正的「融入日常」：你不用打开任何工具，到点它自己运行。

### 三个定时任务（在 Box 里用 scheduled-task-creator 一键创建）

**任务一：每日晚间知识简报**
```
触发：每天 22:00
动作：检查今天所有任务的知识新增 + 阈值状态，生成简报推送企微
效果：睡前一条消息，知道今天积累了什么
```

**任务二：每周一早 9:00 回顾推送**
```
触发：每周一 09:00
动作：生成上周任务进展 + 知识库摘要，推送到企微
效果：周一开工前知道上周做了什么，下周要做什么
```

**任务三：任务停滞提醒**
```
触发：每天 10:00
动作：检查超过 3 天未更新进度的任务，推送企微提醒
效果：不会忘记正在进行的任务
```

### 在 Box 中创建（复制发送）：
```
use_skill("scheduled-task-creator")
```
然后说：
```
帮我创建三个定时任务：
1. 每天晚上 22:00，运行知识库阈值检查，结果推送企微
2. 每周一早上 9:00，生成上周任务回顾摘要，推送企微
3. 每天 10:00，检查超过 3 天未更新的任务，推送企微提醒
```

## 方式四：企微消息触发（最轻量的记录方式）

如果你不想打开任何 IDE，只想在企微聊天时顺手记录：

### 原理
企微 → `wecom_push` MCP 双向打通 → 知识秘书接收消息 → 写入知识库

### 配置后的效果
```
你在企微发给 Box 机器人：
「刚看到一个好方案：用 Bloom Filter 做缓存穿透防护」

Box（知识秘书）自动回复：
「✅ 已记录到 [当前任务]：Bloom Filter 缓存穿透（idea）
  标签：#Bloom Filter #缓存 #防护」
```

手机上随时随地记，完全不需要打开电脑。

## 融入日常的「最小摩擦」原则

| 场景 | 你做什么 | 知识秘书做什么 |
|---|---|---|
| 读文章时 | 正常聊天说「这篇不错」 | 自动存为知识 |
| 写代码时 | 说「卡住了」 | 记录 + 搜知识库 + 推荐 |
| 开会时 | 说「有个想法」 | 自动存，不打断 |
| 睡前 | 什么都不做 | 定时任务推企微简报 |
| 周一开工 | 看企微消息 | 已有上周回顾推送 |
| 导出材料 | 说「生成思维导图」 | 一键生成 HTML 图谱 |

**核心设计哲学：知识秘书不是一个你需要"切换到"的工具，它是你现有工作流的一层透明膜。**


期望输出：
```
✅ 已记录到 [React 管理后台重构]：
   Zustand 替代 Redux 方案（idea）
   标签：#Zustand #Redux #状态管理
```

如果以上都正常，说明已经完全融入你的日常工作流。
