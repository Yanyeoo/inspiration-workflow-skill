# 任务驱动知识库 + 工作流跟踪 Skill v2.0

## 核心理念

**你正在做 N 件事，每件事都有自己的知识库（秘书）。AI 用这些知识帮你判断下一步。**

- 不是「记灵感」→ 而是「这条知识属于哪个任务」
- 不是「单一当前任务」→ 而是「任务列表，随时切换」
- 不是「确认后再存」→ 而是「直接存，轻量反馈」

---

## 数据架构

### 文件结构
```
.workbuddy/memory/
├── tasks.json                # 任务列表（核心）
├── inspirations/
│   └── inspirations.json    # 灵感库（每条绑定 task_id）
└── reviews/                 # 自动压缩生成的综述
    └── review-2026-06-09.md
```

### tasks.json schema
```json
{
  "version": "2.0",
  "active_task_id": "task_001",
  "tasks": [
    {
      "id": "task_001",
      "name": "开发灵感库 Skill",
      "status": "active",
      "progress": 40,
      "created_at": "2026-06-09T11:00:00+08:00",
      "done": false,
      "history": [
        {
          "timestamp": "2026-06-09T11:30:00+08:00",
          "action": "完成 SKILL.md 架构设计",
          "progress": 20
        }
      ],
      "blockers": [],
      "next_steps": ["实现 MVP 脚本", "发 GitHub"],
      "knowledge_count": 6
    }
  ]
}
```

### inspirations.json schema（新增 task_id 字段）
```json
{
  "version": "2.0",
  "entries": [
    {
      "id": "insp_20260609_131804_382160",
      "title": "RAG 优化论文",
      "content": "提出混合检索方案...",
      "type": "paper",
      "tags": ["RAG", "知识库"],
      "task_id": "task_001",
      "created_at": "2026-06-09T12:30:00+08:00",
      "source_url": "https://arxiv.org/...",
      "status": "active"
    }
  ],
  "meta": {
    "total": 1,
    "last_compressed": null,
    "compression_threshold": 50
  }
}
```
> `task_id: null` 表示全局知识，不属于任何任务。

## 触发词

| 场景 | 触发词示例 | AI 动作 |
|---|---|---|
| 切换/新建任务 | 「开始做…」、「新任务…」、「切换到…」 | 创建或切换 active_task |
| 记录知识 | 「我想到…」、「有个想法…」、「这篇论文…」、「链接：…」 | **直接存**，绑定当前任务，轻量反馈 |
| 更新进度 | 「我做了…」、「完成了…」 | 更新 active_task 的 history + progress |
| 遇到困难 | 「卡住了」、「报错…」、「有个 bug」 | 记录 blocker + 用任务知识库推荐 next_steps |
| 问下一步 | 「接下来干嘛」、「下一步是什么」 | 读取 active_task 的 knowledge + next_steps，给出建议 |
| 切换任务 | 「切换到任务 A」、「看看其他任务」 | 列出任务列表，切换 active_task |
| 查看状态 | 「现在进度如何」、「做到哪了」 | 输出 active_task 完整状态（含知识库摘要） |
| 搜索知识 | 「之前关于 RAG 的记录」、「任务 A 的知识库」 | 按任务 + 关键词检索 |
| 导出总结 | 「导出总结」、「生成报告」 | 导出当前任务报告（做了什么 + 知识库 + 下一步） |

## 工作流

### Step 1：任务管理（新增任务 / 切换任务）

**用户说「开始做 X」或「新任务：X」：**
```
AI：「✅ 已创建任务：[开发灵感库 Skill]
    当前活跃任务已切换至此。
    进度：0%  |  知识库：0 条」
```

**用户说「切换到 X」或「看看任务列表」：**
```
AI：「当前任务列表：
    1. [进行中] 开发灵感库 Skill（40%，6 条知识）← 活跃
    2. [进行中] BOX AI 大赛（20%，3 条知识）
    
    要切换哪个？或直接说「开始做 X」创建新任务。」
```

### Step 2：知识捕获（丝滑版，无确认）

**用户说「我想到用 AI 做代码审查」：**

```
AI：「✅ 已记录到 [开发灵感库 Skill] 知识库：
    AI 代码审查（idea）
    标签：#AI提效 #代码审查」
```

**用户分享链接/论文：**
```
用户：「这篇论文有用：https://arxiv.org/...」

AI：「✅ 已记录到 [开发灵感库 Skill] 知识库：
    RAG 优化论文（paper）
    标签：#RAG #检索优化」
```

**知识点自动关联**：当用户连续提到相关话题，AI 在反馈里顺带提示：
```
AI：「✅ 已记录到 [开发灵感库 Skill] 知识库：
    AI 代码审查（idea）
    
    与知识库中「自动化测试框架」（06-05）相关，已自动关联。」
```

### Step 3：工作流跟踪（AI 用知识库辅助决策）

**用户说「卡住了，token 缓存有 bug」：**
```
AI：「✅ 已记录到 [开发灵感库 Skill]：
    困难：token 缓存写入无效数据
    
    根据知识库中的「tai-skill 官方文档」(06-08)，
    推荐下一步：
    1. 检查 tai-auth.sh 第 42 行缓存写入逻辑
    2. 写入前校验 HTTP 状态码（参考知识库已存方案）
    
    当前进度 30% → 要更新吗？」
```

> 关键点：AI **主动读取**当前任务的 knowledge，给出有依据的下一步，而不是泛泛而谈。

**用户说「接下来干嘛」：**
```
AI：「📋 [开发灵感库 Skill] 当前状态：
    进度：40%  |  知识库：6 条
    
    已完成：
    - SKILL.md 架构设计（06-09）
    - 修复 ID 冲突 bug（06-09）
    
    进行中：
    - 实现 MVP 脚本（capture.py 已完成）
    
    根据知识库，推荐下一步：
    1. 发 GitHub（知识库有 gh CLI 操作步骤）
    2. 写参赛文档（知识库有大赛规则链接）
    
    要更新进度还是直接开始？」
```

### Step 4：多任务视图

**用户说「看看所有任务」或「我有哪些任务」：**
```
AI：「📋 任务总览：

    1. [进行中] 开发灵感库 Skill
       进度：40%  |  知识库：6 条
       下一步：发 GitHub、写参赛文档
       
    2. [进行中] BOX AI 大赛
       进度：20%  |  知识库：3 条
       下一步：确定差异化亮点
       
    活跃任务：[1] 开发灵感库 Skill
    输入编号切换，或说「开始做 X」创建新任务。」
```

---

### Step 5：阈值检测与自动压缩

**触发条件**（同 v1，绑定到任务维度）：
- 单个任务的知识库 ≥ **30 条**（可配置）
- 或全局灵感库 ≥ **50 条**（可配置）

**压缩动作**：
1. AI 读取该任务的所有知识
2. 按标签分组，生成**任务知识综述**
3. 存为 `reviews/review-{task_name}-{date}.md`
4. 将已压缩的条目标记为 `archived`

**压缩前询问用户**（唯一的确认步骤，因为压缩是不可逆的结构化操作）：
```
AI：「📦 [开发灵感库 Skill] 知识库已达 30 条，
    要生成知识综述并归档吗？
    
    综述将包含：按标签分组的核心洞察 + 知识关联图」
```

---

### Step 6：导出报告

**用户说「导出总结」：**
```
AI：「📄 正在生成 [开发灵感库 Skill] 任务报告...

    报告包含：
    - 任务概况（进度/时间跨度）
    - 时间线（做了什么，按日期）
    - 知识库摘要（按标签分组）
    - 困难与解决方案
    - AI 推荐下一步
    
    ✅ 已导出到 .workbuddy/memory/exports/task-report-20260609.md」
```

导出格式支持：Markdown（默认，适合 KM/腾讯文档）、JSON、Mermaid 知识图谱。

## AI 行为规则

### ✅ 应该做的

1. **默认直接存**：识别到知识后直接写入，用一行 ✅ 反馈即可，不询问确认
2. **任务绑定**：每条知识必须绑定到任务（当前活跃任务或用户指定），不绑定到任务的存为全局知识（`task_id: null`）
3. **用知识库辅助决策**：推荐下一步时，先读当前任务的知识库，给出有依据的提示
4. **主动关联**：存储时自动检索同任务下的相关条目，在反馈中告知用户（不询问，直接关联）
5. **多任务意识**：随时知道当前活跃任务是哪个，反馈时带上任务名
6. **压缩前确认**：达到阈值后询问用户（唯一需要确认的场景）

### ❌ 不应该做的

1. **不要询问确认**（除压缩外）：「要记录吗？」是糟糕的体验，去掉所有 Y/n
2. **不要强制打断**：用户正在说话时不要插话，等一段话结束再反馈
3. **不要孤立知识**：每条知识都必须有归属（任务或全局），避免「孤儿知识」
4. **不要删除原始数据**：压缩 = 标记 `archived`，不删除

## 配置项

用户可通过对话修改配置：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `compression_threshold` | 50（全局）/ 30（单任务） | 触发压缩的阈值 |
| `auto_detect` | true | 是否自动识别知识（关闭后需手动说「记一下」） |
| `confirm_before_compress` | true | 压缩前是否询问（推荐开启） |
| `reminder_interval` | 10 | 每 N 条知识提醒一次任务进度 |

## 与现有记忆系统的关系

| 现有机制 | 本 Skill 的角色 |
|---|---|
| `YYYY-MM-DD.md`（日记） | 可选：从中提取知识，绑定到对应任务 |
| `MEMORY.md`（长期记忆） | 将压缩后的任务综述核心洞察同步到 MEMORY.md |
| `tasks.json`（新增） | 任务列表，替代原来的单任务 workflow-state.json |

**协同工作**：
- 用户说「记录灵感」→ 写入 `inspirations.json`（带 `task_id`）+ 更新 `tasks.json` 的 `knowledge_count`
- 压缩后 → 综述文档 + 同步核心洞察到 `MEMORY.md`
- 切换任务 → 更新 `tasks.json` 的 `active_task_id`

## 技术实现说明

**MVP 阶段**（本次实现）：
- 使用 JSON 文件存储（`tasks.json` + `inspirations.json`）
- 关键词搜索（不依赖向量数据库）
- 手动触发压缩（不自动）

**未来优化**：
- 接入向量数据库（Chroma / FAISS），实现语义知识检索
- 任务间知识推荐（A 任务的知识推荐给 B 任务）
- 多用户隔离（团队协作场景）
- 与企微/飞书打通，对话即记录
