# 演示脚本 — 录屏指南

> 建议录屏时长：**3~5 分钟**（比赛评委注意力有限）  
> 推荐工具：OBS / 腾讯会议录屏 / macOS 自带录屏  
> 建议分辨率：1920×1080，字体放大到 16px+

---

## 录屏前准备

```bash
# 1. 清空测试数据，从干净状态开始
rm -f ~/.workbuddy/memory/inspirations/inspirations.json
rm -f ~/.workbuddy/memory/tasks.json

# 2. 打开 Box（WorkBuddy）到干净对话页面

# 3. 准备好以下内容备用（复制粘贴，避免录屏时打字出错）
# - 一条灵感文字
# - 一个 arxiv 链接
# - 一个 blocker 描述
```

## 录屏注意事项

| 事项 | 建议 |
|---|---|
| 字体大小 | Terminal 和 Box 均设置 ≥ 16px |
| 鼠标移动 | 慢一点，让观众跟上 |
| 输入速度 | 提前复制好，粘贴输入，避免打字错误 |
| 背景 | 使用深色主题，对比度高 |
| 旁白 | 提前写好稿子，录音清晰；也可以后期加字幕 |
| 剪辑 | 可以剪掉 AI 生成等待的时间，加速处理 |
| 时长 | 控制在 3~5 分钟，超过 5 分钟评委容易跳 |

## 提前造数据脚本（让演示更丰富）

```bash
#!/bin/bash
# demo-data.sh — 预置演示数据

SCRIPT_DIR=~/.workbuddy/skills/inspiration-workflow-skill/scripts

# 创建任务
python $SCRIPT_DIR/workflow.py --new-task "RAG 知识库优化项目"

# 预置几条知识
python $SCRIPT_DIR/capture.py --content "稀疏检索 + 密集检索混合方案" --type idea --tags "RAG,混合检索"
python $SCRIPT_DIR/capture.py --content "https://arxiv.org/abs/2210.11610 RAG 优化论文" --type paper --tags "RAG,论文"
python $SCRIPT_DIR/capture.py --content "BM25 在中文领域效果优于英文 embedding" --type idea --tags "BM25,中文"
python $SCRIPT_DIR/capture.py --content "领域微调：用 triplet loss 对 bi-encoder 做少样本微调" --type decision --tags "微调,bi-encoder"

# 更新进度
python $SCRIPT_DIR/workflow.py --action "完成 BM25 基线实现" --progress 30
python $SCRIPT_DIR/workflow.py --blocker "密集检索模型在领域数据上效果差"

echo "✅ 演示数据准备完成，共 4 条知识，进度 30%"
```
